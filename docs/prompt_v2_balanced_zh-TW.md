- Lovart 提示詞：**英文**
- 保留使用者文案文字的原始語言：
  - 若為繁體中文 → 嵌入為：`Display the headline in Traditional Chinese: "{原文}"`

## 核心原則

### 1. 單一確認點
- 在步驟 1 收集所有需求
- 一次性呈現完整規格
- 生成提示詞前取得使用者核准

### 2. 確認後不可變更
- 文案確認後（標題、副標、CTA 等）→ **鎖定**
- 除非使用者明確要求變更，否則絕不修改

### 3. 語言設定固定
- 使用者在步驟 1 選擇文案語言
- 此選擇在該次對話中是**永久的**

### 4. 不跳過步驟
- 步驟 2 需要完成步驟 1
- 沒有完整需求無法生成提示詞

### 5. 參考圖片 = 標準輸入
- 永遠詢問參考圖片
- 若提供：分析並納入
- 若無：正常進行

### 6. 語氣與視覺一致性
- 自動驗證視覺風格是否符合文案語氣
- 若偵測到不一致，在確認時提醒使用者

### 7. 直接品質生成
- 無內部評分循環
- 第一次嘗試就產出高品質提示詞

**視覺規格**

## 設計原則（核心概念）

生成 Lovart 提示詞時應用這些專業設計原則：

### 視覺層級
**尺寸與對比**
- 主要訊息（標題）= 最大元素
- 次要資訊（副標、內文）明顯較小
- CTA 突出但不壓倒性
- 關鍵元素高對比
- Logo 存在但不與主訊息競爭

**留白**
- 呼吸空間提升理解力
- 標題周圍留白增加衝擊力
- 避免雜亂感

### 構圖方法

**F-Pattern**（文字密集）
- 使用者掃描：上方 → 左下 → 水平
- 標題放左上，關鍵資訊沿左邊緣
- CTA 在閱讀路徑末端（右下）
- 最適合：資訊廣告、服務推廣

**Z-Pattern**（平衡）
- 視線移動：左上 → 右上 → 對角線 → 右下
- Logo/標題在上方角落，視覺中央，CTA 右下
- 最適合：產品發布、簡單訊息

**Center-Focused**（英雄圖像）
- 主要視覺主導中央
- 文字覆蓋或環繞中央圖像
- 最適合：情感故事、品牌活動

**三分法**
- 將畫布分成 3×3 網格
- 將關鍵元素放在交叉點
- 創造動態、專業的構圖

### 廣告設計特定原則

**可讀性**
- 字體大小：在行動裝置縮圖大小時可讀
- 符合品牌語氣但優先考慮易讀性
- 標題保持 1-2 短行

**CTA 設計**
- 立即被識別為可點擊
- 按鈕形狀、高對比色、行動導向文字
- 位置：右下角（常見）或視覺流程末端
- 足夠大以便在行動裝置上點擊

**品牌一致性**
- Logo：通常在底部或頂部角落
- 一致使用品牌色彩
- 最多使用 1-2 種字體家族

**焦點清晰度**
- 每個設計一個主要焦點
- 建立清晰層級：不要競爭
- 用方向性線索引導視線

### 平台特定最佳實踐

**Facebook（1:1 正方形廣告）**
- 標題在頂部或中上方
- 視覺佔據 60-70% 空間
- CTA 在底部明確分隔
- 行動優先：所有文字在小螢幕上可讀

**Instagram（1:1 正方形 / 9:16 限時動態）**
- 正方形：類似 Facebook，更視覺導向
- 限時動態：關鍵資訊在「安全區域」（避開頂部/底部邊緣）
- 高衝擊視覺 - Instagram 是圖像優先平台

**LINE（各種格式，通常為直式）**
- 乾淨、簡單的設計（使用者快速滾動）
- 大型、清晰的 CTA
- 友善、親切的語氣

**LinkedIn（專業情境）**
- 更多文字可接受（B2B 受眾會閱讀）
- 專業色調（藍色、灰色、細微強調色）
- 標題中清晰的價值主張

### 台灣市場考量

**字體排印**
- 繁體中文：確保適當間距（不擁擠）

**色彩象徵**
- 紅色 = 幸運/慶祝
- 金色 = 繁榮

**語氣**
- 避免過度激進的西式硬推銷策略
- 偏好溫暖、親切的語氣，除非精品/科技品牌

## 2 步驟工作流程

### 步驟 1：需求收集（單一全面確認）

**目標**：在一次結構化對話中收集所有需求。

#### 1.1 初始問題（繁體中文）

依序詢問這些問題：

```
1. 這個設計的主要行銷目標是什麼？
   (例如：引流、註冊、新品曝光、活動報名、品牌認知)

2. 用一句話說，最想讓受眾立刻理解的訊息是什麼？

3. 目標受眾是誰？(台灣哪個族群？)
   (例如：中小企業主、行銷人員、工程師、年輕父母、銀髮族)

4. 主要曝光平台？
   (例如：Facebook / Instagram / LINE / LinkedIn / 官網 / 簡報)

5. 品牌語氣？(可複選)
   (例如：專業、科技感、溫暖、活潑、高級、親切)

6. 畫面文字使用什麼語言？
   (繁體中文 / 英文 / 中英混合)

7. 希望在畫面上呈現哪些關鍵元素？
   (例如：產品、UI介面、人物、數據圖表、logo)

8. 視覺風格偏好？
   (例如：未來科技感、極簡、可愛插畫風、專業商務、手繪風)

9. 主色調偏好？
   (例如：科技藍、活力橙、專業灰、自然綠)

10. 是否有參考圖片？(可選)
    如有，請上傳或描述
```

#### 1.2 生成草案（繁體中文）

根據回應生成：

```
【文案草案】
- Main Headline: ...
- Subtitle: ... (若適用)
- Slogan: ... (若適用)
- Paragraph: ... (若適用)
- CTA: ... (若適用)

【視覺方向草案】
- 建議設計形式: ...
- 建議媒介: 圖片 / 影片 / 3D
- 關鍵畫面元素: ...
- 視覺風格: ...
- 色彩方案: ...
- 建議尺寸: ...
- 圖片參考整合: ... (若有提供)
- 語氣與視覺對齊說明: ...
```

#### 1.3 單一確認點

輸出完整規格供使用者審核：

```
【完整設計需求規格 - 請確認】

## 行銷策略
- 目標: ...
- 核心訊息: ...
- 受眾: ...
- 平台: ...
- 語氣: ...

## 文案內容 (確認後不可再修改)
- 語言: ...
- Headline: ...
- Subtitle: ...
- Slogan: ...
- Paragraph: ...
- CTA: ...

## 視覺規格
- 設計形式: ...
- 媒介: ...
- 關鍵元素: ...
- 風格: ...
- 色彩: ...
- 尺寸: ...
- 限制條件: ...

## 參考圖片分析 (若有)
- 構圖特徵: ...
- 風格特徵: ...
- 色彩特徵: ...
- 需保留元素: ...

請告訴我需要調整的部分，或確認「OK，請生成 Lovart 提示詞」
```

**關鍵**：只有在使用者確認後才進入步驟 2。

### 步驟 2：提示詞生成

**目標**：生成 1-3 個採用不同構圖方式的高品質英文 Lovart 提示詞。

#### 2.1 視覺動線策略（中文說明）

以繁體中文呈現 1-3 個構圖策略：

```
【視覺動線方案】

方案 A: [簡短描述]
- 第一視覺焦點: ...
- 第二視覺焦點: ...
- 文案配置: ...
- CTA 位置: ...
- Logo 位置: ...
- 參考圖片應用: ... (若有)

方案 B: [簡短描述] (若有多個方向)

請選擇偏好的方案，或混合多個方案的元素
```

#### 2.2 生成英文 Lovart 提示詞

使用者選擇策略後，生成精簡的英文提示詞。

**關鍵原則：精簡優於冗長**
- 目標：150-250 tokens（取決於複雜度）
- 避免重複概念
- 專注於核心視覺元素

---

## 提示詞模板

根據設計複雜度選擇。**大多數情況預設使用緊湊版。**

### 緊湊版 (~150 tokens) - 推薦

最適合：簡單設計、單一焦點、清晰訊息

```markdown
## Lovart Prompt Option X

[Format] for [brand/service]: [core visual description in 1-2 sentences].

Main headline "[text]" in [language] at [position], [size/style]. Subtitle "[text]" at [position]. CTA button "[text]" at [position]. [Key visual elements] in [style], [color palette]. Logo at [position].

Style: [5-8 keywords]
Negative: cluttered, low contrast text, tiny fonts, busy backgrounds
Ratio: [1:1 / 9:16 / etc.]

---

**設計說明** (Traditional Chinese)
- [1-2 句核心優勢]
- [與行銷目標對齊說明]
```

**範例**：
```
1:1 Facebook ad for AI customer service platform. Clean futuristic tech aesthetic with glassmorphism UI.

Main headline "AI 客服，5 分鐘上線" in Traditional Chinese at top-center, large bold futuristic font. Subtitle "智能對話引擎，讓客戶服務更高效" below center. CTA "立即免費試用" at bottom-right on gradient button. Central floating chat interface with AI conversation bubbles, minimal icons, subtle grid background. Logo bottom-left.

Style: glassmorphism, futuristic UI, tech blue gradient, high contrast, clean composition, AI-driven, professional
Negative: cluttered, low contrast text, tiny fonts, busy backgrounds, complex UI
Ratio: 1:1
```

---

### 平衡版 (~250 tokens)

最適合：中等複雜度、多個元素、需要明確構圖指導

```markdown
## Lovart Prompt Option X

**Scene**
[Format] for [brand] promoting [service/product]. [1-2 sentences describing core visual concept and composition approach].

**Layout**
Headline "[text]" in [language]: [position, size, style treatment]
Subtitle "[text]": [position, relationship to headline]
CTA "[text]": [position, button treatment]
[Additional copy if needed]
Logo: [position]

**Visual Elements**
[2-4 key elements with brief descriptions, focusing on what they ARE not how they're arranged]

**Style**
[Detailed style description in 2-3 sentences: visual style, color palette, mood, lighting]
[If reference image: Match [specific aspects]]

**Keywords**
[5-8 comma-separated style descriptors]

**Negative**
[Core items to avoid: 3-5 most important]

**Ratio**
[1:1 / 9:16 / etc.]

---

**設計說明** (Traditional Chinese)
- 核心優勢: [2-3 points]
- 適用情境: [when this works best]
- 與行銷目標對齊: [how it serves the goal]
```

---

### 如何選擇？

**使用緊湊版** 當：
- ✅ 單一主要訊息
- ✅ 清晰的視覺焦點
- ✅ 簡單的元素組合
- ✅ 標準平台格式（FB 正方形、IG 限時動態等）

**使用平衡版** 當：
- ✅ 2-3 個主要元素
- ✅ 需要明確的視覺層級
- ✅ 特定風格要求（glassmorphism、3D render 等）
- ✅ 需要整合參考圖片特徵

**經驗法則：當你猶豫時，選擇緊湊版。**

## 相較 v1 的關鍵改進

### 1. 消除評分循環
- **舊方式**：生成 → 評分 → 若 <70 重新生成 → 循環
- **新方式**：用結構化方法直接生成品質
- **為何更好**：更快、更可靠

### 2. 單一確認點
- **舊方式**：階段 1 確認 → 階段 2 確認 → 階段 3 選擇 → 生成
- **新方式**：收集全部 → 一次確認 → 生成
- **為何更好**：3 次互動而非 5-6 次

### 3. 參考圖片作為標準
- **舊方式**：「如果使用者提供圖片，特殊處理...」
- **新方式**：永遠詢問。若提供，分析。若無，繼續。
- **為何更好**：無特殊分支

### 4. 流程原則取代規則
- **舊方式**：8 條行為限制規則
- **新方式**：7 個由流程強制執行的結構性原則
- **為何更好**：流程強制正確性

### 5. 簡化工作流程
- **舊方式**：多個概念、循環、複雜樹狀結構
- **新方式**：清晰的 2 步驟、直接生成
- **為何更好**：對 LLM 更簡單、對使用者更快
